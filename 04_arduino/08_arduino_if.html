    If izrazi

    Naše prethodno kolo je vrlo zabavno, ali RGB LED je jako blještava, i trenutno jedini način da je isključimo, jeste da diskonektujemo ceo arduino. Šta mislite da ubacimo jedno dugme, kojim bismo mogli da isključimo / uključimo dovod struje na ovu diodu kako nama odgovara?

    Obratite pažnju da nam dve nogice dugmeta nisu bitne, i mogu da se nalaze i u drugoj polovini proto pločice, onoj na koju ništa nije povezano. Od dve bitne nogice, jedna treba da bude povezana na uzemljenje, a druga treba da ima prvo vezu ka pinu 3 (žuta žičica) a takođe i vezu sa izvorom napona od 5V (crvena žičica na cvenoj šini), preko otpornika.

    Sada možemo pristupiti modifikaciji kôda iz prethodne lekcije:

    File Edit Sketch Tools Help
    Sketch_jun03a
    int crvena = 11;
    int zelena = 10;
    int plava = 9;

    void setup() {
    }

    void loop() {
    analogWrite(crvena, random(0, 255));
    analogWrite(zelena, random(0, 255));
    analogWrite(plava, random(0, 255));
    delay(1000);
    }
     
     

    Prvo ćemo dodati varijablu koja će se odnosti na pin 3, s kojim je naše dugme povezano:

    File Edit Sketch Tools Help
    Sketch_jun03a
    int crvena = 11;
    int zelena = 10;
    int plava = 9;

    int dugme = 3; // dodato

    void setup() {
    }

    void loop() {
    analogWrite(crvena, random(0, 255));
    analogWrite(zelena, random(0, 255));
    analogWrite(plava, random(0, 255));
    delay(1000);
    }
     
     

    Sada treba da proverimo da li je dugme pritisnuto. Po defaultu, pull up otpornik na arduinu, drži napon na ovom pinu na HIGH. Kada se dugme pritisne, odnosno dok je dugme pritisnuto, napon prelazi na LOW. Čim pustimo dugme, napon se vraća na HIGH. Ono što mi želimo da uradimo, jeste da uvedemo još jednu varijablu, sada bool tipa, koja će da menja svoju vrednost svaki put kad napon na pinu 3 (dugme) postane LOW.

    Da bismo ovo postigli, koristićemo if blok. If blokovi se pišu na sledeći način:

    if (uslov) {
    // kôd koji će se izvršavati svaki put kada je "uslov" jednak true
    // ako je uslov jednak false, ceo if blok će biti preskočen
    }

    Uslov može da bude ili neka varijabla tipa bool, ili logički izraz koji će evaluirati ili na true, ili na false.

    Logički izrazi mogu koristiti logičke operatore. U sledećem izrazu ćemo videti operator ekvivalencije ==. Ovaj operator proverava da li su izrazi sa leve i desne strane jednaki. Vodite računa da ne pišete = u ovim evaluacijama, jer je = operator koji dodeljuje vrednost, dok == proverava da li su dve vrednosti jednake. Naravno, evaluacija vraća true ako su izrazi jednaki, odnosno false ako se na bilo koji način razlikuju.

    File Edit Sketch Tools Help
    Sketch_jun03a
    int crvena = 11;
    int zelena = 10;
    int plava = 9;
    int dugme = 3;

    bool sija = false; // dodato

    void setup() {
    }

    void loop() {
    if (digitalRead(dugme) == LOW) {
    sija = !sija;
     }

    analogWrite(crvena, random(0, 255));
    analogWrite(zelena, random(0, 255));
    analogWrite(plava, random(0, 255));
    delay(1000);
    }
     
     

    Bool tip podataka može da ima samo dve vrednosti, true ili false. Operator ! je logički operator negacije, koji true pretvara u false, i obrnuto. Ono što ovaj if blok kaže jeste: Ako je dugme pritisnuto, promeni vrednost sija varijable. Sada još samo treba da kažemo arduinu da ako je vrednost varijable "sija" true, da pušta struju na RGB LED, i menja mu boje na svaku sekundu. To ćemo postići pomoću još jednog if bloka, kome će uslov sada biti naša varijabla "sija":

    File Edit Sketch Tools Help
    Sketch_jun03a
    int crvena = 11;
    int zelena = 10;
    int plava = 9;
    int dugme = 3;
    bool sija = false;

    void setup() {
    }

    void loop() {
    if (digitalRead(dugme) == LOW) {
    sija = !sija;
     }

    if(sija) {
    analogWrite(crvena, random(0, 255));
    analogWrite(zelena, random(0, 255));
    analogWrite(plava, random(0, 255));
    delay(1000);
    }
    }
     
     

    Kako smo ovo napisali, naša LED neće sijati kad pokrenemo skeč (bool sija = false;) ali kada jednom pritisnemo dugme, vrednost sija će se promeniti na true, i LED će početi da svetli, menjajući boje na svaku sekundu. Međutim, ako ponovo pritisnemo dugme, LED će nastaviti da sija. Potrebno je da kažemo arduinu šta da radi i kada se vrednost sija ponovo vrati na false. Za to ćemo koristi else blok. Else blok nema uslov, već se izvršava onda kada uslov iz if bloka nije ostvaren. Drugim rečima, ili će se izvršavati kod iz if bloka, ili onaj iz else bloka.

    File Edit Sketch Tools Help
    Sketch_jun03a
    int crvena = 11;
    int zelena = 10;
    int plava = 9;
    int dugme = 3;
    bool sija = false;

    void setup() {
    }

    void loop() {
    if (digitalRead(dugme) == LOW) {
    sija = !sija;
     }

    if(sija) {
    analogWrite(crvena, random(0, 255));
    analogWrite(zelena, random(0, 255));
    analogWrite(plava, random(0, 255));
    delay(1000);
    }

    else {
    analogWrite(crvena, 0);
    analogWrite(zelena, 0);
    analogWrite(plava, 0);
    delay(1000);
    }
    }
     
     

    digitalRead() funkcija očitava digitalnu vrednost zadatog pina, dakle ili HIGH ili LOW, kako su to jedine vrednosti koje digitalni pinovi mogu imati. Svi pinovi su po defaultu input pinovi, tako da kod digitalRead i analogRead funkcija, nije neophodno da se prvo koristi pinMode(). PinMode() se mora pozivati jedino u slučaju digitaWrite() funkcije. Koderi generalno upisuju pinMode() u set-up i za digitalni input i analogni output, naprosto da bi bili pregledno na jednom mestu, šta se sve od pinova koristi za šta.

    Pored if i else bloka, imamo i else if blok, kojim možemo zadati dodatni uslov. Dakle izvršavaće se kada uslov iz if bloka nije ispunjen, ali uslov iz else if bloka jeste ispunjen. Ako posle if i else if bloka dodamo else blok, kôd u njemu će se izvršavati kada ni uslov iz if bloka, ni uslov iz else if bloka nisu ispunjeni.

    Vaš zadatak će sada biti lakši od prikazanog primera, vežbe radi, povežite običan LED sa dugmetom, i namestite ga da se pali / gasi pritiskom na dugme.

    Operatori poređenja

    U if uslivima, često vrše poređenja kako bi se utvrdilo da li if blok treba da se izvršava ili ne. U poređenjima figurišu operatori poređenja u koje pored == ekvivalencije, spadaju još i != nije jednako, < manje od, > veće od, <= manje ili jednako, i >= veće ili jednako.

    Logički operatori

    Logički operatori se koriste u situacijama kada želimo da proverimo više uslova u isto vreme. Ovde spadaju && poznat kao "i" operator, koji će davati true, ako su oba uslova ispunjena. Kod && operatora se prvo poredi prvi izraz, i tek ako je on istinit, proverava se i drugi izraz. Logičko ili se označava sa ||, i kod njega je potrebno da samo jedan od zadativ uslova bude ispunjen, da bi ceo izrat bio true. Prvo se evaluira prvi izraz, a na evaluaciju drugog izraza se prelazi tek ako je prvi netačan. Operator negacije ! koji pretvara true u false i obrnuto, takođe spada u ovu kategoriju.
