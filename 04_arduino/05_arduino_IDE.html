    Arduino IDE

    Ok, šta je en: Integrated Development Environment, sr: Sveobuhvatno kodersko okruženjeIDE? Zašto IDE a ne software? IDE jeste software, ali takav da je napravljen za kodiranje. Od običnog tekstualnog editora se razlikuje po čitavom nizu funkcionalnosti, prilagodljivosti, i sklonosti da u velikoj meri pomaže koderima.

    Pre nego što počnemo, promenite pin koji vrši napajanje vašeg kola, na jedan od digitalnih pinova. Ovim će vaše kolo privremeno prestati da radi, ali nema brige.

    Pokrenite arduino IDE. U meniju, pod File, kliknite na new, ili samo pritisnite Ctrl+N na tastaturi. Otvoriće vam se novi osnovni skeč, koji će izgledati ovako:

    File Edit Sketch Tools Help
    Sketch_may30a
    /* komentar */
    void setup() {
    // put your setup code here, to run once:

    }

    void loop() {
    // put your main code here, to run repeatedly:

    }
     
     

    Ono što ovde vidimo, su dve funkcije, setup() i loop(). Reč void znači da ove dve funkcije ne vraćaju ništa. Kada funkcije nešto vraćaju, int, string, bool, potrebno je da se završavaju sa return izrazom koji vraća podatak očekivanog tipa. Reč void se koristi samo kod deklaracija funkcija, i označava da ove funkcije ne vraćaju nikakav podatak funkciji koja ih je pozvala.

    Kôd koji upišete u setup() će se izvršiti samo jedared, a kôd koji upišete u loop() će se odvijati iznova i iznova, u nedogled.

    Ovakav templejt ćete videti svaki put kad pokrenete novi skeč.

    Ukucajte sada:

    File Edit Sketch Tools Help
    Sketch_may30a
    /* komentar */
    void setup() {
    pinMode(12, OUTPUT);
    }

    void loop() {
    // put your main code here, to run repeatedly:

    }
     
     

    Broj 12 ovde, treba da korespondira sa pinom koji ste povezali sa svojom proto pločom kao napajanjem - ako niste koristili pin 12, ili upišite odgovarajući broj pina, ili prebodite kolo.

    Ono što ova komanda radi, jeste da inicira pin sa tim brojem, kao output. Ako bismo umesto OUTPUT, ovde upisali INPUT, time bismo ovaj pin inicirali da očitava vrednost koja dolazi na ovaj pin. Ova vrednost se može podesiti i na INPUT_PULLUP, što će postaviti otpornik između naše pločice i ulaznog signala, pa se ova komanda preporučuje kod iščitavanja radi zaštite arduina. Hajde sada da pustimo struju na ovaj pin.

    File Edit Sketch Tools Help
    Sketch_may30a
    /* komentar */
    void setup() {
    pinMode(12, OUTPUT);
    }

    void loop() {
    digitalWrite(12, HIGH);
    }
     
     

    Funkcija digitalWrite prima 2 parametra, prvi je pin na koji želimo da pustimo struju, a drugi parametar je efektivno bool vrednost, koja može biti ili HIGH (true) ili LOW (false). Ako upišemo HIGH, struja će teći pod naponom od 5V ili 3,3V u zavisnosti od pločice, ako upišemo LOW, struja će biti obustavljena, odnosno napon će biti 0V.

    Sada možemo da pokrenemo naš program. Klinkite na i za koji trenutak, struja će ponovo poteći kroz naš arduino, i naša dioda će zasvetleti.

    Pre nego što uploudujete svoj skeč na arduino, potrebno je da u meniju pod Tools / Board odaberete svoj model pločice (Arduino uno). Ako to ne uradite, IDE neće moći da naše vaš arduino i neće moći da uplouduje skeč, nego će vam davati džumbus grešaka u konzoli. Druga stvar koju je potrebno da uradite, jeste da u meniju pod Tools / Port, izaberete COM.
    Ako sada zakomentarišete pinMode() funkciju u setup() funkciji, i ponovo pokrenete program klikom na , videćete da će dioda sada svetleti znatno slabije. To je otuda što kada se ne podesi pinMode na željeni pin, digitalWrite() funkcija sama odobrava interni PULLUP otpornik koji suzbija velike struje. Nakon što ste ovo testirali, slobodno odkomentarišite pinMode() funkciju.

    Sada verovatno deluje jasno šta treba da uradimo da bi dioda treptala?

    File Edit Sketch Tools Help
    Sketch_may30a
    /* komentar */
    void setup() {
    pinMode(12, OUTPUT);
    }

    void loop() {
    digitalWrite(12, HIGH);
    digitalWrite(12, LOW);
    }
     
     

    Deluje logično, zar ne? Međutim ako opet pokrenete program, LED će i dalje neprekidno svetleti. Zašto to? Razlog je taj što se kôd izvršava vrlo brzo, i LED trepće daleko brže nego što je ljudsko oko u stanju da to registruje, pa nam izgleda kao da kontinualno sija. Hajde da dodamo neki delay() kako bi usporili treptanje dovoljno da ga možemo videti. Delay se definiše u mili sekundama, tako da ako hoćete da svetloda trepće na svaku sekundu, potrebno je dodati delay(1000);. Da bi treptala na svakih pola sekunde delay(500) i tako dalje.

    File Edit Sketch Tools Help
    Sketch_may30a
    /* komentar */
    void setup() {
    pinMode(12, OUTPUT);
    }

    void loop() {
    digitalWrite(12, HIGH);
    delay(1000);
    digitalWrite(12, LOW);
    delay(1000);
    }
     
     

    Sada ste već na pola puta do novogodišnjih svetiljki.

    Ovde još vredi napomenuti, da svaka arduino pločica, ima na sebi integrisanu jednu svetlodu, koja može da se koristi kao izlaz umesto našeg strujnog kola, ili paralelno s njim. Ova dioda je vezana za jedan od pinova, označena je slovom L i verovatno možete pogoditi za koji je pin vezana na osnovu toga pored kog se pina nalazi. Međutim, nema potrebe da pogađate, jer postoji ključna reč koja trigeruje baš ovaj pin. Ta reč je LED_BUILTIN, koju biste u gornjem kôdu koristili umesto broja pina.

    File Edit Sketch Tools Help
    Sketch_may30a
    /* komentar */
    void setup() {
    pinMode(LED_BUILTIN, OUTPUT);
    }

    void loop() {
    digitalWrite(LED_BUILTIN, HIGH);
    delay(1000);
    digitalWrite(LED_BUILTIN, LOW);
    delay(1000);
    }
     
     
    Vaš zadatak je da podesite ovo kolo da trepće ove dve diode naizmenično. Kada dioda u našem kolu svetli, integrisana treba da bude isključena, i obrnuto, kada integrisana dioda svetli, naša dioda treba da bude isključena.
