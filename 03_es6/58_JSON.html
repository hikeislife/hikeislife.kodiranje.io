    JSON

    Postoji malo nesuglasica oko izgovora, ali Daglas ga zove DžejsonJSON je najkorišćeniji format za prenos podataka sa servera ka klijentu. Iako ime sugeriše da je ekskluzivan za javaskript, zapravo se koristi u svim programskim jezicima. JSON fajlovi su laki, ljudski čitljivi, i nezamenjiv alat. Danas čak omogućavaju i klijentskom Javaskritu, da simulira funkcionalnost bekend aplikacije kada se koristi u tzv Javasript Api MarkupJAM staku. Pre JSON-a uglavnom je bio korišćen XML, koga i danas možete sporadično videti, ali JSON dominira ako ništa drugo ono otuda što sadrži manje krša od xml fajlova.

    Krokford je osmislio JavaScript Object NotationJSON format 2001-e godine. Fajl nije ništa drugo do običan JS koji može sadržati i samo jedan podatak i biti legitiman JSON fajl, ali pošto je to vrlo nepraktično, imati ceo fajl za samo jedan podatak, to JSON obično dolazi kao jedan Array, u kome je svaki član, jedan JS objekat, i svi ti objekti se odvajaju zarezima. Za razliku od klasičnih JS objekata, u JSON-u, i ključevi i vrednosti moraju biti stringovi:

    [
    {
    "ime" : "pera",
    "prezime" : "mitić",
    "ključ" : "vrednost",
    "godine" : 48,
    "davitellj" : true
    }
    ]

    Brojevi i bulijani u JSON-u se ne pišu pod navodnicima. Brojevi mogu biti pozitivni ili negativni, decimalni, ili čak naučna notacija poput 2.9e8. Pored ovih osnovnih podataka, JSON podržava i null, Array-e, ili druge objekte, koji onda takođe moraju biti formatirani kao JSON, a u biti predstavljaju granjanje strukture.

    Podaci u JSON-u ne mogu biti undefined, funkcije, ili datumi kao Date objekti. Ako treba da pošaljete datum, možete ga poslati kao string, a onda ga naknadno konvertovati u Date objekat.
    JSON je vrlo osetljiv i format mora da bude u celosti tačan. Jedan propušteni zarez ili zarez viška, jedna izostavljena zagrada, ili zagrada viška i ceo fajl će prestati da radi. Ako koristite apostrofe umesto navodnika, JSON neće raditi. JSON fajlovi znaju da budu jako veliki. Zato, ili ga pišite pažnjivo, ili koristite generatore.
    Krokforda vole često da predstavljaju kao prvog čoveka koji je otkrio da javaskript ima dobre delove. Sam Krokford kaže da je dugo odbijao da nauči JS, jer mislio da je jezik glup. Ali kako je morao da se nateže s njim, u nekom trenutku je shvatio da će morati da ga nauči. I tada je promenio mišljenje. Njegova knjiga, "Javaskript, dobri delovi", nije selekcija retkih stvari koje su dobre u javaskriptu, već je selekcija stvari koje je Javaskript uradio drugačije od drugih jezika, a koje su dobra rešenja koje Krokford smatra da svi novi jezici treba da implementiraju.

    Za rad sa JSON formatom, javaskript koristi JSON objekat, koji ima samo tri metode, stringify, parse i toSource. toSource je nestandardna metoda implementirana samo u nekim brauzerima.

    Parsiranje

    Kada se JSON šalje preko web-a, on se šalje kao string koji obično nazivamo JSON string. Razlog za ovo je taj što se JSON šalje pomoću en: Hyper Text Transfer Protocolhttp-a, koji može da transportuje samo text, odnosno samo stringove. Stringom bi bilo mnogo teže manipulisati, i mnogo teže pristupiti željenim podacima nego u objektu. Zato taj string prvo parsiramo i time ga konvertujemo nazad u JS objekat (ili array objekata)

    const beogradskiDavitelji = `[
    {
    "ime" : "pera",
    "prezime" : "mitić",
    "ključ" : "vrednost",
    "godine" : 48,
    "oženjen" : false
    },
    {
    "ime" : "spiridon",
    "prezime" : "kopicl",
    "ključ" : "vrednost",
    "godine" : 28,
    "oženjen" : false
    }
    ]`
    const davitelji = JSON.parse(beogradskiDavitelji);

    Nakon što smo isparsirali ovaj string u JSON, možemo lako pristupiti pojedinim članovima ovog array-a korišćenjem odgovarajući indexa, npr

    console.log(davitelji[0]);

    >{
    "ime" : "pera",
    "prezime" : "mitić",
    "ključ" : "vrednost",
    "godine" : 48,
    "oženjen" : false
    }

    Takođe možemo pristupiti i pojedinim vrednostima tog objekta:

    console.log(davitelji[0].ime);

    > "pera"

    Stringifikacija

    Ako imamo običan JS objekat, kod koga su ključevi bez navodnika, i koji koristi apostrofe, možemo ga konvertovati u JSON string, pomoću JSON.stringify() metode. Tako ga možemo i poslati preko web-a kao npr informaciju koju vraća neki API, ili ga možemo adlje konvertovati u JSON pomoću JSON.parse() metode.

    const mojObjekat = {
    ime : 'pera',
    prezime : 'mitić',
    ključ : 'vrednost',
    godine : 48,
    oženjen : false
    }

    const mojString = JSON.stringify(mojObjekat);

    console.log(mojString);

    > "{\"ime\":\"pera\",\"prezime\":\"mitić\",\"ključ\":\"vrednost\",\"godine\":48,\"oženjen\":false}"

    Ovde možete videti da su svi navodnici u JSON stringu izbegnuti pomoću \. Ovo ne treba da vas brine, ako sada na mojString pozovete JSON.parse() dobićete očekivani rezultat i kose crte će nestati.

    Ukoliko polazni JS objekat sadrži datum, on će biti konvertovan u string. Ako polazni objekat sadrži neke metode, one će biti u celosti uklonjene iz JSON stringa.

    JSON Schema

    JSON skima, ili šema, je nešto kao meta data o JSON fajlu, i takođe dolazi kao JSON fajl. Šema sadrži podatke o tome koje parametre naš JSON fajl, i koji im je tip. Koristi se za validaciju JSON fajlova, pri čemu se u jednom koraku vrši poređenje sa zadatom šemom, a u sledećem koraku semantička analiza JSON fajla.
