    Prototipsko nasleđivanje

    Nasleđivanje je jedna izuzetno bitna karakteristika objekata. Nemate pojma koliko se forsiraju četiri stuba objektnog programiranja, AEIP, gde je jedan od ta četiri stuba, upravo en: Inheritancenasleđivanje.

    U prethodnoj lekciji smo videli da kad imamo konstruktorsku funkciju:

    function Davitelj(ime, prezime) {
    this.ime = ime;
    this.prezime = prezime;

    this.davi = function() {
    console.log('Ne volite karanfile, eto vam sad...');
    }
    }

    I pomoću nje konstruišemo objekat:

    const davitelj = new Davitelj('Pera', 'Mitić');

    Taj objekat stiče svojstvo __proto__ koje je zapravo prototip naše konstruktorske funkcije. Međutim, ako ste zagledali šta konzola sve ispisuje, možda ste zapazili da prototip funkcije Davitelj, takođe ima svoj prototip:

    Davitelj.prototype.__proto__;

    // Vraća Object objekat

    Ovde ne možemo pozivati Davitelj.prototype.prototype, jer je Davitelj.prototype objekat, a objekti nemaju prototype svojstvo. Ali možemo pozvati:

    davitelj.__proto__.__proto__;

    I dobićemo isti rezultat. Ako ponovo pozovemo .__proto__:

    davitelj.__proto__.__proto__.__proto__;

    // Dobićemo null

    Dakle, jedno .__proto__ vraća prototip konstruktorske funkcije, dva .__proto__ vraćaju Object, a Object nema prototip, pa treće .__proto__ vraća null. Po defaultu, svi objekti u JS-u nasleđuju od Object. Ovaj niz objekata koje dobijamo dodajući svaki put još po jedno .__proto__, nazivamo en: chain of inheritancelanac nasleđivanja ili prototipski lanac. Ako za bilo koji objekat u JS-u idemo uz lanac nasleđivanja, uvek ćemo na kraju završiti sa null, najčešće odmah nakon što stignemo do Object objekta.

    Kako da kreiramo JavaSkript nasleđivanje po svojoj meri

    Naš davitelj objekat dakle nasleđuje od Object objekta. Ali šta ako hoćemo da nasleđuje od nekog našeg objekta recimo mama objekta?

    function Mama() {
    //
    }

    Mama.prototype.maminSin = function() {
    console.log('Odakle nam davitelj? Rodila ga mama!');
    }

    Sada možemo da postavimo Mamu kao prototip na našeg Davitelja

    Davitelj.prototype = Object.create(Mama.prototype)

    Šta ovo znači? Pa, sad svi naši davitelji, imaju mamu, i dostupni su im sva svojstva i sve metode sa mama prototipa.

    const davitelj = new Davitelj('Pera', 'Mitić');

    davitelj.maminSin();

    Ovde koristimo Object.create() jer tako ne dolazi do poziva ka Mama() funkciji, sve što se dešava jeste da Object.create() postavlja Mamu u prototipski lanac Davitelj-a. Da smo koristili ključnu reć new, to bi pozvalo Mama funkciju, a pošto u ovom trenutku samo želimo da kreiramo vezu između mame i davitelja, nije nam potrebno da pozivamo mamu. Nama je potrebno da pozovemo Mamu tek kad kreiramo instancu Davitelja, jer tada možemo da dodelimo argumentima iz mame, vrednosti karakteristične za našeg trenutnog davitelja. Da bismo pozvali Mamu u trenutku kad kreiramo novu instancu davitelja, dodaćemo:

    function Davitelj(ime, prezime) {
    Mama.call(this); // Dodato
    this.ime = ime;
    this.prezime = prezime;

    this.davi = function() {
    console.log('Ne volite karanfile, eto vam sad...');
    }
    }

    Sada možemo i da kastumizujemo šta će podjedini davitelj da govori svojoj mami, sve što treba jeste da namestimo da funkcija Mama prima neki argument, i onda izgovori taj argument kad se pozove metoda maminSin():

    function Mama(poštapalica) {
    this.poštapalica = poštapalica || 'Odakle nam davitelj? Rodila ga mama!';
    }

    Mama.prototype.maminSin = function() {
    console.log(this.poštapalica);
    }

    Treba da modifikujemo i davitelja tako može da prosledi poštapalicu mami:

    function Davitelj(ime, prezime, poštapalica) {
    Mama.call(this, poštapalica); // Dodato
    this.ime = ime;
    this.prezime = prezime;

    this.davi = function() {
    console.log('Ne volite karanfile, eto vam sad...');
    }
    }
    Davitelj.prototype = Object.create(Mama.prototype);

    Sada možemo kreirati objekat Davitelj:

    spira = new Davitelj('spiridon', 'kopicl', 'bejbi bejbi')

    Na kome možemo pozvati metodu spira.maminSin(), i ona će vratiti Spirinu poštapalicu, "bejbi, bejbi", što deluje kao da ima smisla, jer je Spiridon SPOILER: na kraju oženio svoju mamu

    Prototipski lanac u JavaSkriptu

    Ako sada pokušamo da korišćenjem __proto__ metode vidimo prototipski lanac našeg davitelja, rezultati će biti vrlo zbunjujući. Naime, spira.__proto__ vraća mamu. To je otuda što kada smo podesili prototip za našeg davitelja sa:

    Davitelj.prototype = Object.create(Mama.prototype)

    Time smo ujedno podesili i da konstruktor našeg objekta davitelj bude Mama. JS zna da je spira Davitelj, što se može videti na osnovu:

    spira instanceof Davitelj;

    // Vraća true

    Ako hoćemo da ispravimo ovo ponašanje, da nam konstruktor za spiru bude davitelj, to možemo ispraviti ovako:

    Davitelj.prototype.constructor = Davitelj;

    Sada možemo lepo da sagledamo proto lanac davitelja:

    spira.__proto__; // Vraća Davitelj
    spira.__proto__.__proto__; // Vraća Mama
    spira.__proto__.__proto__.__proto__; // Vraća Object
    spira.__proto__.__proto__.__proto__.__proto__; // Vraća null

    Zaključak

    Sve u svemu, kada hoćemo da izgradimo prototipski lanac, sve o čemu treba da vodimo računa su sledeća tri reda kôda:

    Mama.call(this, poštapalica);
    // Koji poziva sledeći konstruktor u lancu, zatim

    Davitelj.prototype = Object.create(Mama.prototype)
    // Koji dodaje sledeći prototip u lanac, i

    Davitelj.prototype.constructor = Davitelj;
    // Koji koriguje konstruktora

    I to je sva filozofija prototipskog nasleđa :)
