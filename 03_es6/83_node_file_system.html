    File Sistem modul

    Rad sa fajlovima, čitanje iz fajlova, upisivanje u fajlove, predstavlja osnovu bekend aplikacija.

    U prethodnoj lekciji, kreirali smo osnovnu strukturu konzolne aplikacije, ali sada nam je potrebno da podatke koje korisnik unese, sačuvamo u JSON fajl kako bi kasnije pri ponovnom pokretanju aplikacije, prethodni unosi i dalje bili dostupni. JSON fajl predstavlja array objekata, gde će svaki objekat, predstavljati poseban unos.

    Da bismo ovo postigli, koristićemo jedan od kôr nôd modula, fs, što je skraćenica od file system. Ovaj modul omogućava nôdu da radi sa fajlovima, što je kao što znate, bila jedna od stvari koje javaskript po dizajnu nije mogao da radi.

    const fs = require('fs')

    Ovim smo uključili fs modul u naš fajl i sada ga možemo koristiti pozivanjem različitih metoda na fs objektu. fs može da radi samo sa stringovima, a nama treba JSON, tako da ćemo morati da radimo konverzije.

    U opštem slučaju, ne želimo da nam app.js fajl bude zatrpan suvišnim kodom, pa ćemo za početak kreirati novi fajl, recimo unosi.js, u koji ćemo uneti funkcionalnost koja će se koristiti u app.js fajlu.

    U ovom novom fajlu, kreiraćemo funkciju koja će izvršavati dodavanje unosa u fajl:

    const fs = require('fs')

    dodajUnos = (pod) => {
    console.log("unos dodat");
    }

    module.exports = {
    dodajUnos
    }

    Sada treba da uključimo ovaj novi fajl unosi.js u app.js fajl:

    const unosi = require('unosi')

    I naravno, u komandi dodaj treba da modifikujemo handler metodu tako da poziva dodajUnos funkciju:

    yargs.command({
    command: "dodaj",
    describe: "Dodaj nov unos",
    builder: {
    unos: {
    describe: "tekst za unos",
    demandOption: true,
    type: "string"
    }
    },
    handler: (argv) => {
    unosi.dodajUnos(argv)
    }
    })

    fs.writeFile() i fs.writeFileSync()

    fs.writeFile i fs.writeFileSync obe primaju dva parametra, prvi je ime fajla u koji želimo da upišemo tekst, kao string, a drugi argument je šta tačno hoćemo da upišemo, takođe string. writeFile je asinhrona, što će reći neće blokirati izvršavanje ostatka kôda dok čeka da upiše tekst, a writeFileSync je sinhrona, i ako dođe do bilo kakvog zastoja kod upisa, to će se odrazii na sav ostali kôd. S druge strane, ljudima je mnogo lakše da shvate tok izvršavanja kôda kad se on izvršava sinhrono, red po red, nego kad je asinhron. Sva mudrost asinhronosti jeste da se takav kôd izmešta na zasebnu nit, i ostatak kôda nastavlja da se izvršava nevezano za tu metodu. Kada ostatak kôda odradi svoje, tek onda se vraća odgovor o tome da li je asinhrona funkcija završila. U našem slučaju je nebitno koju bismo koristili, ali u opštem slučaju je bolje koristiti asinhrone funkcije, jer http zahtevi znaju da budu spori, i ne želimo da cela aplikacija zakuca dok čeka da se u fajl upiše red teksta. Mi ćemo u ovom primeru koristi writeFileSync jer u suprotnom bismo morali da definišemo šta treba da se odradi kada funkcija završi upisivanje teksta.

    dodajUnos = (pod) => {
    fs.writeFileSync('unosi.json', pod.unos);
    }

    Drugi parametar writeFile i writeFileSync funkcije, je takođe string, ali ovde je taj string u varijabli, iz kog razloga nije u navodnicima. dodajUnos prima objekat pod, koji je zapravo argv objekat koji smo prosledili iz app.js fajla kroz handler metodu. Onda biramo svojstvo .unos na tom pod objektu, i upisujemo ga u fajl.

    Ako nismo kreirali fajl unosi.json, nôd će ga kreirati za nas, i u njega upisati vrednost iz pod.unos. Ako fajl već postoji, ova funkcija će pregaziti sav sadržaj tog fajla, i upisati samo trenutni sadržaj pod.unos varijable.

    Takođe, ovako prosleđen pod.unos, će se upisati kao običan tekst, a nama je potrebno da se unese kao JSON kako bismo mogli kasnije da manipulišemo tim podatkom.

    Za početak, da bismo izbegli gaženje prethodnih unosa novim unosom, koristićemo drugu metodu fs objekta, metodu koja "čita" fajl:

    fs.readFile i fs.readFileSync

    Kao u slučaju fs.writeFile i fs.writeFileSync, ove dve metode rade istu stvar, otvaraju fajl i iz njega vraćaju sadržaj, fs.readFile asinhrono, a fs.readFileSync, sinhrono. I sada, u opštem slučaju želite da koristite fs.readFile jer ona nema blokirajuću prirodu. A i ime joj je kraće za kucanje. Međutim i sada je mnogo jasnije šta se dešava u kôdu sa sinhronom metodom pa ćemo s njom i raditi.

    Pošto ćemo u opštem slučaju, čitati sadržaj našeg fajla više puta, kada upisuje novi unos, kada ih ispisujemo, pa i kada brišemo jedan od unosa, to ćemo kôd koji čita fajl, izmestiti u zasebnu funkciju, koju onda možemo pozivati iznova i iznova.

    vratiSveUnose = () => {
    const bafer = fs.readFileSync('unosi.json');
    }

    Treba naglasiti, da kada fs.readFile ili readFileSync vrate sadržaj fajla, on neće doći ni kao string ni kao JSON, već će nam se vratiti u vrlo nečitkom binarnom formatu koji se naziva buffer. Da bismo iz binarnog fajla dobili string, koristićemo .toString() metodu, a onda ćemo parsirati taj string, kako bismo dobili mnogo korisniji oblik, json.

    vratiSveUnose = () => {
    const bafer = fs.readFileSync('unosi.json');
    const str = bafer.toString();
    const podaci = JSON.parse(str);

    return podaci;
    }

    Takođe, kako smo napomenuli gore u slučaju writeFile, ako prosleđeni fajl ne postoji, writeFile će ga kreirati. Međutim, ako fajl unosi.json ne postoji u slučaju readFile funkcija, moramo napisati kôd koji će znati kako da se snađe s takvom situacijom. Za takve situacije, koristimo try/catch blok:

    vratiSveUnose = () => {
    try {
    const bafer = fs.readFileSync('unosi.json');
    const str = bafer.toString();
    const podaci = JSON.parse(str);

    return podaci;
    }
    catch (e) {
    return [];
    }
    }

    Ovakav kôd će vraćati sadržaj fajla unosi.json, ako unosi.json postoji i ako je u pitanju validan JSON. Ako ta dva uslova nisu ispunjena, try blok će prestati da se izvršava, i umesto njega će se izvršiti catch blok, koji u ovom slučaju vraća prazan Array.

    Sada možemo da se vratimo na našu funkciju dodajUnos(), koja na prvom mestu treba da primi ono što vratiSveUnose() vraća:

    dodajUnos = (pod) => {
    const podaci = vratiSveUnose()
    }

    Ono što sada treba da uradimo, jeste da formatiramo argument pod koji dodajUnos prima, i da ga dodamo na ovaj Array, bilo da je on prazan, ili ne:

    dodajUnos = (pod) => {
    const podaci = vratiSveUnose()

    const podatak = {
    unos: pod.unos
    }

    podaci.push(podatak)
    }

    Sada smo spremni da sačuvamo ove izmene, a kako bismo u opštem slučaju čuvali podatke u više navrata, to je najbolje da i taj deo kôda izdvojimo u zasebnu funkciju:

    unesiIzmene = (unosi) => {
    const zaUnos = JSON.stringify(unosi);
    fs.writeFileSync('unosi.json', zaUnos)
    }

    Konačno, još samo treba da pozovemo funkciju unesiIzmene() iz funkcije dodajUnos(), i imamo jednu od 4 osnovne CRUD ili REPL operacije odrađene:

    dodajUnos = (text) => {
    const podaci = vratiSveUnose()

    const podatak = {
    unos: text.unos
    }

    podaci.push(podatak)

    unesiIzmene(podaci);
    }

    U ovom slučaju, mi smo i dalje pregazili celokupan sadržaj fajla, novim sadržajem, ali smo sada postigli da novi sadržaj fajla obuhvata i stare i novu vrednost u isto vreme, tako da je krajnji efekat, kao da smo samo dopisali novu vrednost ne dirajući stare unose.
