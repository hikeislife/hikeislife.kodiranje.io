    Brojevi

    Kod nas u Srbiji, kao i u većini sveta, uobičajeno je da se decimalna mesta odvajaju zarezom , a da se tačke koriste za grupisanje cifara. Ipak, kako su većinu kompjuterskih jezika osmislili amerikanci, to su oni primenili, svoju, u svetu jedinstvenu notaciju, tako da se u programskim jezicima decimale odvajaju tačkom ..

    Većina programskih jezika ima čitav niz različitih vrsta brojeva, intidžere, floating points (plutajuće tačke), decimalne, duple... pa onda još u zavisnosti od toga koliko prostora zauzimaju u memoriji, dodatne podkategorije. Sada mora da vam je drago što ste odlučili da učite JS, jer JS ima samo jedan tip broja, i taj se zove: broj. Number. U pozadini, svi brojevi u javaskriptu su tipa plutajuće tačke. Pa, ne može sve da valja. Jedan broj da vlada svima, super ideja, ali da taj jedan broj bude upravo IEEE-754 standard koji ne radi kako treba, e to je već suboptimalno.

    Kako ne radi kako treba, pitate se vi? Evo, banalan primer, ukucajte u konzolu:

    0.2 + 0.1

    // vraća 0.30000000000000004
    Ovo može da bude ozbiljan problem kada se radi sa recimo novcem, pa je česta praksa da se recimo umesto dinara, računa u parama, i da se onda pare konvertuju u dinare kada je potrebno prikazati broj na ekranu. Para nema decimale, pa neće ni da ispoljava greške plutajuće tačke.
    IEEE-754 standard definiše kako čuvati brojeve u naučnoj notaciji, kao 2.9e8, u binarnom formatu. Greške u zaokruživanju se javljaju usled toga što je neke brojeve nemoguće precizno prikazati u svakom sistemu. Recimo u ljudskom, dekadnom sistemu, ne možete da prikažete 1/3, ona evaluira na 0.333333333... i te trojke nastavljaju da se ponavljaju, zauvek. Mi ljudi, možemo da kažemo, "o, to je 0.33, i 3 * 0.33 je 1" umesto da tvrdimo da je 3 * 0.33 zapravo 0.99. U binarnoj notaciji, decimalno 0.1 je 0.0 0011 0011 0011... i to 0011 se ponavlja u nedogled. Kompjuteri ne razumeju ovo ponavljanje i nedogled, i to pravi grešku.

    Operatori

    Standardni peratori za rad sa brojevima se nazivaju aritmetički operatori i ponašaju se isto kao što ste navikli u matematici:

    Simbol Opis
    ** Stepenovanje, npr 27 postaje 2**7
    * Množenje
    / Deljenje
    + Sabiranje
    - Oduzimanje
    % Moduo, ostatak pri deljenju, npr 1 % 2 daje 1.
    m % n => m - (~~(m / n) * n)
    gde ~~(m / n) predstavlja celobrojni rezultat deljenja
    ++ Inkrement: n++ => n + 1
    -- Dikrement: n-- => n - 1

    Operacije imaju isti prioritet kao u matematici. Prvo se vrši stepenovanje, zatim množenje i deljenje, i tek na kraju sabiranje i oduzimanje. Upotrebom običnih zagrada () može se promeniti redosled operacija gde će se prvo obaviti operacije u zagradama, pa tek onda operacije van zagrada. Takođe, zagrade se mogu koristiti i da grupišu određeni deo računa kako bi bio pregledniji.

    Moduo % se recimo koristi da se utvrdi da li je broj paran ili neparan. n % 2 vraća 1 za neparne brojeve, i 0 za parne brojeve, gde 1 onda evaluira kao true (tačno) i 0 evaluira kao false (netačno)
    Inkrementi i++ i dikrementi i-- se koriste često u petljama i rekurzijama gde se vrednost brojača i u svakom krugu povećava za 1.

    Pored ovih, koristimo i operatore za dodelu vrednosti:

    Simbol Opis
    = m = n, dodeli vrednost n m-u
    **= m **= n => m = m**n
    *= m *= n => m = m * n
    /= m /= n => m = m / n
    += m += n => m = m + n
    -= m -= n => m = m - n
    %= m %= n => m = m % n

    Ono što se ovde dešava je sledeće. Program će da uzme trenutnu vrednost m, i da izračuna rezultat s desne strane izraza, npr m * n, a onda će taj rezultat da dodeli levoj strani izraza, m-u, ili kraće, samo m *= n

    let m = 5;
    // vraća 5

    m *= 2;
    // vraća 10

    m *= 2;
    // vraća 20

    Pored ovih, postoje i en: Bitwisebitski operatori, koji rade sa bitovima:

    Simbol Opis
    & AND, samo dve jedinice daju jedinicu:
    3 & 1 => 0011 & 0001 => 0001 => 1
    | OR, bar jedna jedinica daje jedinicu
    3 | 1 => 0011 | 0001 => 0011 => 3
    ~ NOT, jedinice postaju nule, i obrnuto
    ~3 => ~0011 => 1100 => -4
    ^ XOR, samo jedna jedinica daje jedinicu
    3 ^ 1 => 0011 ^ 0001 => 0010 => 2
    << levi šift, insertuje zadati broj 0 s desne strane
    3 << 1 => 0011 << 1 => 0110 => 6
    >> desni šift, uklanja zadati broj bitova s desne strane
    3 >> 1 => 0011 >> 1 => 0001 => 1

    Ovi operatori konvertuju brojeve u 32-o bitne brojeve, izvrše operaciju nad njima, i vrate ih u dekadni sistem.

    Number objekat i JavaSkript brojevi kao objekti

    Ako u svom brauzeru otvorite konzolu, i ukucate samo reč Number, veliko N, konzola će vam reći da je u pitanju funkcija. Ovo je tzv konstruktorska funkcija i može se pozvati sa:

    Number();

    Konstruktorske funkcije kreiraju nove instance objekata, i u ovom slučaju se kreira objekat tipa Number. Ako ukucate:

    const testBroj = new Number();
    typeof testBroj;

    // vraća Object

    const praviBroj = 5;

    testBroj == praviBroj // vraća false
    testBroj === praviBroj // vraća false
    Dakle, broj kreiran pomoću konstruktora, neće evaluirati jednak tom istom broju, jer nisu istog tipa. Zato se ne preporučuje korišćenje konstruktora za kreiranje primitivnih tipova podataka.
    Svrha Number objekta je u tome da za razliku od objekata, primitivne vrednosti nemaju metode. Da bi moglo da se radi sa primitivnim vrednostima, JS privremeno "obmotova" primitivne u wrapper objekat kako bi mogao na njih da primeni metode. Iako iz ovakvih objekata može da se kreira novi broj, to ne treba raditi, ne samo iz razloga evaluacije podataka, nego i otuda što je i vama i kompjuteru koji treba da izvrši ovaj kôd, lakše i brže da naprosto dodelite brojnu vrednost, tzv literal, varijabli nego da to radite preko ključne reči new i konstruktorske funkcije.

    JavaSkript Number objekat, definiše niz propertija i metoda koje su dostupne za rad sa brojevima.

    Metode na objektima mogu biti statičke, koje su dostupne bez instanciranja, i prototipske, koje su dostupne na instancama. U slučaju JS Number objekta, najzanimljivije su nam prototipske metode:

    toExponential()
    toFixed()
    toLocaleString()
    toPrecision()
    toString()
    valueOf()

    Statičke metode dostupne u Number objektu su:

    isFinite()
    isInteger()
    isNaN()
    isSafeInteger()
    parseFloat()
    parseInt()

    Propertiji u Number objektu su:

    EPSILON: 2.220446049250313e-16
    MAX_SAFE_INTEGER: 9007199254740991
    MAX_VALUE: 1.7976931348623157e+308
    MIN_SAFE_INTEGER: -9007199254740991
    MIN_VALUE: 5e-324
    NEGATIVE_INFINITY: -Infinity
    NaN: NaN
    POSITIVE_INFINITY: Infinity

    Operacije s brojevima u JavaSkriptu

    Broj metoda na Number objektu je vrlo mali, za ozbiljnije operacije s brojevima, koristi se Math objekat

    Prototipske metode

    Prototipske metode su one koje, budući da su definisane na prototipu objekta, bivaju nasleđene nad svim instancama tog objekta, tako da se za razliku od statičkih metoda, ove ne moraju pozivati sa imenom objekta Number. Ove metode na Number objektu se najviše i koriste, pa ćemo njih prve obraditi.

    Pozivaju se tako što se iza varijable, iza samog broja, ili pak iza druge metode koja vraća broj, stavi tačka, pa naziv metode. Iza metode onda idu zagrade, u koje se upisuju parametri. Zagrade su obavezne čak i kada ne prosleđujemo parametre:

    broj.metoda(parametar);
    Kada ove metode pozivamo direktno nad brojevima koji nemaju decimale, npr 555.metoda(), kompajler će bacati grešku:
    SyntaxError: identifier starts immediately after numeric literal

    Ova greška se javlja otuda što kompajler ne zna da li tačka predstavlja decimalnu tačku, ili poziv metode. Da bismo je izbegli, možemo ili broj staviti u zagrade: (555).metoda(), ili staviti jedno prazno mesto iza broja a pre tačke: 555 .metoda().

    Konvertovanje broja u string .toString()

    Metoda može da primi jedan parametar i to tzv radix ili bazu za broj, te je u pitanju ceo broj koji mora biti u opsegu od 2 do 36.

    (555).toString(11);

    // Vraća "465"

    465 je 555 u brojnom sistemu kome je osnova 11 umesto 10. Ako se radix izostavi, vratiće broj u dekadnom sistemu, kao string, dakle '555'.

    Zaokruživanje brojeva .toFixed()

    Ova metoda može da primi jedan parametar i to je broj od 0 do 20. Ovaj broj predstavlja broj decimala na koliko želimo da izvršimo zaokruživanje. Ona vraća zaokružen broj, kao string.

    5.7356.toFixed(2);

    // Vraća "5.74"

    Ako se broj željenih decimala izostavi, smatraće se da je 0:

    5.7356.toFixed();

    // Vraća "6"

    Zaokruživanje brojeva znači da će se brojevi iza zadatog broja cifara odbaciti, i da će se zaostale cifre zadržati u svom prethodnom obliku ako je prva odsečena cifra manja od 5 odnosno broj će se povećati ako je prva odsečena cifra 5 ili veća. Ako hoćemo samo da odbacimo cifre i da nam ostane neizmenjen broj bez obzira na to šta smo odsekli, treba nam trunkcija. Math objekat ima metodu Math.floor() koja ima takav efekat, odnosno i 2.3 i 2.7 će postati 2. Ipak, najefikasniji način da se trunkcija postigne je malo konfuzan na prvu, a svodi se na primenu bitskih operatora. Ono što bitski operatori rade jeste da odbace sve decimale, i onda izvrše svoju operaciju. Recimo bitska negacija ~ će odbaciti sve decimale, a onda okrenuti sve bitove. Ako se bitska negacija ponovi, svi bitovi se vraćaju u prethodno stanje, a decimale i dalje nedostaju. Npr ako želimo da dobijemo celobrojni rezultat deljenja:

    const celobrojnoDeljenje = ~~(9/6);

    // Vraća 1

    Zagrade su ovde bitne, jer će se bitski operator primeniti pre deljenja pa bez zagrada ne bismo postigli željeni rezultat.

    Definisanje preciznosti brojeva .toPrecision()

    I ova metoda vraća broj kao string, sa zadatom preciznošću. Željena preciznost se može proslediti kao parametar u zagradama. Za razliku od .toFixed() koja će vratiti broj sa zadatim brojem decimala, .toPrecision() vraća zadati broj cifara, uključujući tu i celobrojni deo broja.

    2.53.toPrecision(1);
    // Vraća "3"

    2.53.toPrecision(2);
    // Vraća "2.5"

    2.53.toPrecision();
    // Vraća "2.53"

    Ako se kao željena preciznost prosledi broj veći od ukupnog broja cifara, biće dodate nule. Ako se prosledi broj manji od broja cifara u celobrojnom delu, vratiće se broj u tzv naučnoj notaciji:

    (299792458).toPrecision(1);

    // Vraća "3e+8"

    Ova metoda ne prihvata 0 kao parametar, i baca grešku opsega

    (299792458).toPrecision(0);

    // baca: RangeError: precision 0 out of range

    Prikaz brojeva u naučnoj notaciji .toExponential()

    I ova metoda vraća broj kao string, a može da primi jedan parametar (od 0 do 20) i on bi predstavljao broj decimala:

    (299792458).toExponential(4)

    // Vraća "2.9979e+8"

    Ako joj se prosledi vrednost koja nije brojna vrednost, metoda vraća NaN.

    Prikaz brojeva u zadatom lokalitetu .toLocaleString()

    Ova metoda prima parametar koji predstavlja željeni lokal. Lokal za nas bi bio "sr" a imao bi efekat da zameni tačku s zarezom, odnosno da grupiše cifre po tri i razdvoji ih tačkom:

    2.9979e+8.toLocaleString("sr");

    // Vraća "299.790.000"

    Vrednost broja u Number objektu .valueOf()

    Od svih proto metoda, ova je jedina koja ne vraća string već broj. Primenjuje se na instancu Number objekta a vraća njegovu vrednost:

    const brObj = new Number(5);
    const br = brObj.valueOf();
    typeof br;

    // Vraća number

    Statičke metode za rad s brojevima u JavaSkriptu

    U programiranju, kad se kaže da je metoda statička, to znači da nije potrebna instanca klase da bi se metoda pozvala, nego se može pozvati preko same klase. JavaSkript nema klase, bar ne u onom smislu u kome klase postoje u drugim jezicima. Tako da se ove metode u javaSkriptu i ne mogu pozivati preko klasa, ali se pozivaju preko oih osnovnih objekata, kao što je u našem slučaju Number objekat.

    Number.metoda(vrednost)

    Provera da li je neka vrednost broj .isNaN

    Postoje dve verzije isNaN() i nisu međusobno jednake. Jedan isNaN je na globalnom window objektu, i samim tim mu se pristupa preko isNaN(vrednostKojuProveravamo). U tom slučaju dakle ne mora da se kuca ime objekta s koga se isNaN() poziva. Ova metoda proverava da li je neki broj NaN ili ne, i takođe da li je neki podatak broj ili ne. Ako je podatak broj, metoda vraća false, a ako nije broj, vraća true.
    isNaN(Number);

    // vraća true

    Number je objekat, samim tim nije broj, pa metoda vraća true

    Ovo nije ono što želimo od metode koja se zove isNaN, ne želimo da vraća true za objekte već samo za NaN vrednosti.

    Drugi isNaN se nalazi na Number objektu, dakle pristupa mu se preko Number.isNaN(). Ovaj isNaN, proverava da li je posmatrana vrednost NaN ili nije, uzimajući u obzir da tip podatka mora biti broj:
    Number.isNaN(Number);

    // vraća false

    Globalno isNaN se susreće s ovim problemom jer je bila dizajnirana tako da pre evaluacije pokuša da konvertuje zadatu vrednost u broj.

    Polifil
    Number.isNaN = Number.isNaN || function(value) {
    return value !== null && (value != value || +value != value);
    }

    Provera da li je broj konačan .isFinite

    Čita se iz-fajnajt. Znam da se neki muče s ovim izgovorom, eto sad znate.

    Ova metoda proverava da li je broj broj i da li je konačan, odnosno da nije beskonačan. Vraća true za konačne brojeve i false za beskonačne.

    Number.isFinite(5/0);

    // vraća false
    Postoji i globalna isFinite metoda, koja slično globalnoj isNaN metodi, prisiljava vrednost koju evaluiramo u broj, i stoga daje pogrešne rezultate. Koristite Number.isFinite() umesto isFinite()

    Provera da li je broj ceo broj .isInteger

    Metoda Number.isInteger() proverava da li je prosleđena vrednost ceo broj ili ne.

    Number.isInteger(5/2)

    // vraća false

    Provera da li je broj siguran u opsegu IEEE-754 .isSafeInteger

    Sigurni celi brojevi po IEEE-754 standardu se nalaze u opsegu od -(253 -1) do 253 -1. Brojevi u tom opsegu se mogu prikazati pomoću standarda, i ni jedan broj koji se ne može predstaviti, neće biti zaokružen na naš broj. Npr broj 253 može da se prikaže, ali broj 253 + 1 ne može, i zaokružuje se na 253. Odnosno, 253 i 253 + 1 se prikazuju na isti način u IEEE-754 standardu, tako da ni jedan od njih ne spada u sigurne intidžere.

    Number.isSafeInteger(2**53);

    // vraća false

    Konvertovanje stringa u ceo broj .parseInt()

    Ova metoda prima dva parametra, string koji je potrebno parsirati, i osnovu u kojoj se broj koga treba parsirati nalazi, a vraća ceo broj. Kad se kaže osnova misli se na to da li je broj binaran: 2, dekadan: 10, hexadecimalan: 16 i sl:

    Number.parseInt('101a', 2);
    // Vraća 5

    Number.parseInt('101a', 5);
    // Vraća 26

    Number.parseInt('101a', 16);
    // Vraća 4122

    Number.parseInt('101a', 10);
    // Vraća 101

    Metoda parsira onaj deo stringa koji ima smisla da bude broj. Iz toga razloga je slovo a zanemareno prilikom parsiranja u svim slučajevima osim kada je osnova 16, jer u hexadecimalnom, a je broj.

    Konvertovanje stringa u decimalan broj .parseFloat()

    Metoda prima string i parsira validan deo u decimalni broj:

    Number.parseFloat('156');

    // Vraća 156

    Svojstva na Number objektu u Javaskriptu

    Propertiji Number objekta su statički pa im se pristupa uvek preko Number.ime_propertija a ne preko Number objekta koji smo sami kreirali.

    Ako na primer hoćemo da pristupimo EPSILON propertiju, to radimo na sledeći način:

    Number.EPSILON

    S tim u vezi, ovo nisu vrednosti kojima ćete često pristupati, ove vrednosti su karakteristike specifikacije. Navodim ovo samo zato što objekte još nismo pokrili.

    NaN od Not a Number ili "nije broj". NaN je definisan i u okviru window objekta, pa mu se ne mora psitupati preko Number.NaN. window je globalni objekat, i svim parametrima u njemu se može pristupiti samo navođenjem imena parametra bez navođenja imena objekta. Dakle,

    window.NaN
    NaN
    Number.NaN

    // svi vraćaju NaN

    NaN je numerička vrednost koja je rezultat matematičke operacije koja nije uspeala. 0/0 kao nevalidan broj je i dalje broj, vrednost koja ne predstavlja validan broj, radije nego da js daje grešku, daje NaN. NaN nije JS stvar, zapravo je definisan u IEEE-754 floating point standardu da postoji preko 16miliona različitih vrednosti floating pointa koje su NaN - iz tog razloga, vrlo je mala verovatnoća da je jedan NaN zaista jednak drugom NaN-u.

    NaN === NaN

    // uvek vraća false

    Ranije je u JS-u, jedini način da se pouzdano utvrdi da li je neka vrednost NaN, bio da se ta vrednost uporedi sama sa sobom i ako nije jednaka sama sebi, onda jeste NaN. Postojala je globalna funkcija isNaN, koja nije bila pouzdana, za utvrđivanje da li je neka vrednost NaN ili ne. Kasnije je na Number objekat uvedena joj jedna isNaN funkcija koja zaista proverava da li je broj NaN ili nije.

    POSITIVE_INFINITY i NEGATIVE_INFINITY su zanimljivi jer ne postoje u većini drugih jezika. Definisani su i na globalnom objektu (window) kao čita se infinitiInfinity i -Infinity. Možete dobiti Infinity u konzoli deljem bilo kog broja sa 0.

    Rezultat
    n * Number.POSITIVE_INFINITY
    n > 0, uključujući Infinity    		 Infinity
    n * Number.POSITIVE_INFINITY
    n < 0, uključujući -Infinity    		 -Infinity
    n / Infinity
    n > 0    		 0
    n / Infinity
    n < 0    		 -0
    0 * Infinity NaN
    NaN * Infinity NaN
    Infinity / n
    n > 0    		 Infinity
    Infinity / n
    n < 0    		 -Infinity
    Infinity / Infinity NaN
    Infinity / -Infinity NaN

    EPSILON je apsolutna vrednost razlike između broja 1, i najmanjeg broja koji je veći od 1 koji se može prikazati standardom.

    MAX_SAFE_INTEGER je 253 - 1. Ovo je najveći ceo broj s kojim JS može da radi, ili kako to kaže ime broja, najveći siguran, bezbedan broj. Šta to zapravo znači? Ako radite s brojevima većim od ovog, JS će da daje nebulozne rezultate:

    const siguran = Number.MAX_SAFE_INTEGER;
    const plus1 = siguran + 1;
    const plus2 = siguran + 2;
    const plus10 = siguran + 10;

    console.log(plus1);
    // vraća 9007199254740992

    console.log(plus2);
    // vraća 9007199254740992 isto kao plus1

    console.log(plus10);
    // vraća 9007199254741000

    console.log(plus1 === plus2);
    // vraća true

    console.log(plus10 - plus1);
    // vraća 8 umesto očekivanog 9

    Analogno, MIN_SAFE_INTEGER je -253 - 1 i označava najmanji intidžer s kojim se može pouzdano računati.

    MAX_VALUE i MIN_VALUE su najveći i najmanji broj respektivno, koji se mogu prikazati u JS-u. Number.MAX_VALUE je 21024 i dodavanjem brojeva na MAX_VALUE nastavlja da vraća Number.MAX_VALUE.
