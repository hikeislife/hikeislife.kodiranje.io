    Array metode ES1

    Array spada u složene tipove podataka, a u JS-u svi složeni tipovi podataka su zapravo samo objekti, i kao takvi imaju niz dostupnih metoda. Ako u konzolu ukucate samo reč Array (veliko A) i enter, videćete spisak svih metoda dostupnih nad Array-ima. Takođe ćete videti i da konzola tvrdi da je Array funkcija?!? A ja vam upravo rekoh da je Array objekat? Ko tu nema pojma? Ako u konzoli kreirate neki Array arr (koristite literal), i potom pozovete typeOf arr, dobićete potvrdu da je Array ipak objekat. Funkcija koju konzola pominje kada ukucate Array, jeste konstruktorska funkcija, koja generiše objekat, i to u ovom slučaju objekat koji je Array. Ako ukucate sada u konzolu Array() (sa zagradicama) pozvaćete ovu konstruktorsku funkciju, i dobićete kao odgovor prazan Array, koga je ova konstruktorska funckija upravo kreirala.

    Ne postoji nikakav smislen razlog da se metode dostupne nad Array-ima u JS-u, podele prema verziji jezika s kojom su uvedene. Tj, takva podela ima jednako smisla kao i podela po azbučnom redu. Ipak, pošto ih ima mnogo, valja ih nekako "slajsovati" u lekcije lakše za varenje, i onda vreme kad su metode uvedene može delovati kao dodatna zanimljivost. To što su metode na ovoj strani uvedene sa ES1, ništa ne znači, i one i dan danas rade isto kao i prvog dana. Opet će na vrhu strane biti navedene one metode koje se više koriste, dok će one manje zanimljive biti potisnute na dno strane. S tim u vezi, veliki broj metoda iz ove generacije se koristi jako često

    Određivanje dužine niza .length

    Length je svojstvo svakog niza i predstavlja njegovu dužinu kao celi broj njegovih elemenata.

    const mojArray = [1, 2, 3];

    mojArray.length;

    Pošto .length nije metoda već en: propertysvojstvo, poziva se bez (). Length će uvek vratiti broj za jedan veći od maksimalnog indexa Array-a.

    Sada kada znamo kako da dobijemo dužinu Array-a, možemo da dobijemo i poslednji član tog Arraya, ma koliko da je dugačak:

    mojArray[mojArray.length -1];

    Ovim možemo definisati i metodu koja bi uvek vraćala poslednji član Array-a:

    function last(x) {
    return x[x.length - 1];
    }

    last(mojArray);

    Dodavanje elemenata u JavaSkript niz .push() i .unshift()

    Kada želimo dodati element u Array, to uvek možemo postići tako što bismo upisali novi element, na prazan index:

    mojArray[5] = "Pera Mitić"

    Ovo deluje dovoljno jednostavno, osim što onda moramo da vodimo računa o vrednosti indexa, i ako kojim slučajem dodamo element na previše veliki index, kreiraćemo solidan broj praznih mesta u našem Array-u. A iz perspektive izvršavanja kôda, Array-i sa praznim mestima su najsporiji. Brauzeri gotovo da i ne pokušavaju da optimizuju kôd koji se obavlja nad Array-ima sa praznim mestima. Array-i s praznim mestima su nešto što po svaku cenu hoćemo da izbegnemo. Plus ako prosledimo previše mali index, pregazićemo već postojeći element novim elementom

    Umesto toga, imamo dve metode, koje vode računa o indexu umesto nas.

    Dodavanje elementa na kraj niza .push()

    push() će uvek dodati novi element na sam kraj našeg Array-a.

    const arr = [7, 2, 9, 0, 6];

    arr.push(11);

    // Rezultira sa [7, 2, 9, 0, 6, 11];
    // Novi array je duži

    Dodavanje elementa na početak niza .unshift()

    unshift() radi slično što i push(), samo što dodaje nov element na početak Array-a. Isto kao i push, vraća novu dužinu Array-a:

    const arr = [7, 2, 9, 0, 6];

    arr.unshift(11);

    // Rezultira sa [11, 7, 2, 9, 0, 6];
    // Novi array je duži
    Unshift ne radi u IE8 i ranijim verzijama.

    Uklanjanje elemenata sa JavaSkript niza .pop() i .shift()

    Suprotno push i unshift metodama, pop i shift uklanjaju element iz Array-a.

    Uklanjanje poslednjeg elementa niza .pop()

    pop() uklanja poslednji element Array-a, i vraća taj element.

    const arr = [7, 2, 9, 0, 6];

    arr.pop(); // Ovde ne prosleđujemo nikakav parametar

    // Rezultira sa [7, 2, 9, 0];
    // Vraća 6, poslednnji element array-a

    Uklanjanje prvog elementa niza .shift()

    shift() uklanja prvi element Array-a, i vraća taj element:

    const arr = [7, 2, 9, 0, 6];

    arr.shift(); // Ovde ne prosleđujemo nikakav parametar

    // Rezultira sa [2, 9, 0, 6];
    // Vraća 7, prvi element array-a
    Ove 4 metode, push(), pop(), shift() i unshift(), sve menjaju dužinu polaznog Array-a. Ukoliko vam je bitno da zadržite originalni Array tokom izvršavaja kôda, obavezno ga za kloniranje vidi slice metoduklonirajte pre nego što počnete s njim da radite.

    Sortiranje elemenata niza u JavaSkriptu .sort()

    Ova metoda menja polazni Array. Sort reorganizuje polazni Array po azbučnom redu, od A do Z. Drugim rečima, ako primenite sort() na Array brojeva, dobićete nezadovoljavajuće rezultate:

    const arr = [5, 25, 7, 3, 8, 41];
    arr.sort();

    // Vraća [25, 3, 41, 5, 7, 8]
    Šta se ovde zapravo dešava? Kako JS uopšte uspeva da sortira brojeve po abecednom redu? Pre nego što započne sortiranje, JS konvertuje svaki član Array-a u string, i onda poredi karakter po karakter stringa, tj, njihovu unicode code point vrednost sa ostalim članovima Array-a. Kao rezultat toga, 2 dolazi pre 5, ali i 25 dolazi pre 5, na isti način kako "a" i "ana" oba dolaze pre "banana". Ako vam zatreba da znate code point vrednost nekog karaktera, možete koristiti string metodu: string.codePointAt(0), kao na primer "8".codePointAt(0) (vraća 56).

    Ova metoda, može međutim da primi funkciju kao argument, i ta funkcija treba da definiše kako želimo da izvršimo sortiranje. Ova komparativna funkcija, može da vrati pozitivan broj, negativan broj, ili 0. Ako funkcija vraća negativan broj, sort() zadržava trenutni redosled elemenata, a ako vrati pozitivan broj, zameniće im mesta. Zatim prelazi na sledeća dva elementa. Npr, da bismo sortirali brojeve od najmanjeg ka najvećem, koristićemo:

    const arr = [5, 25, 7, 3, 8, 41];
    arr.sort(function(a, b){return a-b});

    // Vraća [ 3, 5, 7, 8, 25, 41 ]

    A ako hoćemo da sortiramo brojeve opadajućim redosledom, koristićemo:

    const arr = [5, 25, 7, 3, 8, 41];
    arr.sort(function(a, b){return b-a});

    // Vraća [ 41, 25, 8, 7, 5, 3 ]

    sorting algoritam

    Čim se pomene sortiranje, prirodno se nameće i pitanje koji algoritam JS koristi za sortiranje? Pa, ovo zapravo ne zavisi od javascript-a, nego od js motora. EcmaScript standard ne sugeriše koji algoritam treba da se koristi, tako da je ovo prepušteno proizvođačima brauzera na volju. Tako npr, SpiderMonkey koristi kombinaciju merge sort algoritma (sortiranje spajanjem) i insercionog algoritma (sortiranje umetanjem) za kraće Array-e, dok V8 koristi kombinaciju kviksorta i insercionog sorta.

    Ovrtanje redosleda elemenata JavaSkript niza .reverse()

    .reverse() menja redosled svih elemenata Array-a, tako da onaj koji je bio nulti postaje poslednji, a onaj koji je bio poslednji, postaje nulti.

    mojArray.reverse();

    Ako recimo želimo da sortiramo Array stringova od z do a, koristili bismo: mojArray.sort().reverse()

    Odsecanje i kloniranje JavaSkript nizova .slice()

    Kao što ime sugeriše, ova metoda otseca deo Array-a i vraća taj odsečak. Može da primi 2 parametra, ali ne mora da primi nijedan.

    Ako se metodi prosledi jedan parametar, taj parametar će biti index od kog želimo da izvršimo odsecanje. Odsecanje će se izvršiti do poslednjeg elementa Array-a. Ako prosledimo negativan broj, index odsecanja se broji od poslednjeg elementa. Odsecanje se i dalje vrši ka poslednjem elementu Array-a. Pošto se index 0 odnosi na prvi element Array-a a 0 ne može biti negativna, to poslednji element Array-a ima index -1

    Ako navedemo oba parametra, odsecanje se vršiti od prvog navedenog indexa, do poslednjeg, ne računajući poslednji.

    const mojArray = [1,2,3,4,5,6,7,8,9];
    const mojSkraceniArray = mojArray.slice(4,7);

    // mojSkraceniArray će biti: [ 5, 6, 7 ] (indexi 4, 5 i 6)

    Kloniranje JavaSkript Array-a

    Pošto je Array referentni tip podataka, što će reći, pojedini članovi Array-a se čuvaju na hîpu, a samo se adresa čuva na šteku, to nije moguće klonirati Array prostim:

    const mojArray = [1,2,3,4,5,6,7,8,9];
    const mojNeuspeliKlon = mojArray;

    mojNeuspeliKlon === mojArray
    // Vraća true

    Ono što smo ovim postigli jeste da smo varijabli mojNeuspeliKlon, dodelili istu vrednost sa šteka, koja je samo adresa ka hîpu, te da obe varijable sad gledaju na istu memorijsku lokaciju na hîpu. Ako sada izmenimo mojNeuspeliKlon na bilo koji način, u isto vreme ćemo izmeniti i mojArray. Ova dva Array-a, su zapravo jedan isti Array, samo sa dva aliasa.

    Da bismo zaista klonirali array, u smislu, duplirali vrednosti na hîpu, i dali oboma Array-a različite adrese na šteku, možemo koristiti slice() metodu bez argumenata. Kako ona vraća novi Array umesto da modifikuje stari, taj novi Array će dobiti novu alokaciju na hîpu, i time novu adresu na šteku, i efektivno dobijamo potpuno novi Array. Ako sada izmenimo taj novi Array, originalni Array ostaje neizmenjen.

    Insertovanje jednog JavaSkript niza u drugi .splice()

    Splice bi se mogao prevesti kao uplitanje, prespajanje. Kada prespojite pokidane žice, kaže se da ste ih splajsovali. Bio je i onaj, valjda horor film, u kome su splajsovali ljudski gen sa genom neke životinje, i kreirali čudovišnu devojčicu, koja ih je pretpostavljam sve pobila na kraju. Ne znam, mrzelo me da gledam taj film, možda se nešto drugo desilo na kraju. splice() metoda, isto tako upliće dva Array-a u jedan, mada su rezultati manje čudovišni.

    Ovo uplitanje može da bude čisto dodavanje, ili dodavanje sa uklanjanjem, pri čemu metoda vraća uklonjeni deo Array-a, menjajući pri tom polazni Array.

    Prvi parametar ove metode je obavezan, i predstavlja index na kome želimo da umetnemo nove elemente. Drugi parametar je opcion, i govori koliko elemenata želimo da uklonimo, počev s zadatim indexom. Ako se prosledi 0, neće se ukloniti ni jedan element.

    Treći i ostali parametri, su elementi koje je potrebno dodati u array počev s pozicijom zadatog indexa:

    const mojArray = ["Pera", "Mila", "Laza"];
    mojArray.splice(1, 0, "Žika", "Sima");

    // mojArray će sada biti: [ "Pera", "Žika", "Sima", "Mila", "Laza" ]

    Ako se elementi za dodavanje ne proslede splice metodi, ona će samo izbaciti zadati broj elemenata iz polaznog Array-a.

    Konvertovanje JavaSkript niza u string .toString()

    Vraća string u kome se nalaze svi elementi Arraya-a, razdvojeni zapetama.

    Konvertovanje JavaSkript niza u string sa zadatim separatorom .join()

    Join slično toString() metodi, konvertuje Array u string, samo je sada moguće odabrati simbol kojim želite da razdvojite pojedine članove Array-a. Ako se separator ne navede, po defaultu će se koristiti ,. Npr:

    const arr = ["mama", "tata", "brat", "kravata"];
    arr.join(" i ");

    // Vratiće string: "mama i tata i brat i kravata";

    Konkatinacija JavaSkript nizova .concat()

    .concat() objedinjuje dva Array-a u jedan novi, dakle početni Array-i ostaju neizmenjeni.

    const voće = ["banana", "jabuka", "šljiva"];
    const povrće = ["luk", "krompir", "krastavac"];

    const zaKupovinu = voće.concat(povrće);

    // Vraća zaKupovinu = [ "banana", "jabuka", "šljiva", "luk", "krompir", "krastavac" ]

    .valueOf()

    Ova metoda vraća sam Array.

    mojArray.valueOf();

    // vraća mojArray
