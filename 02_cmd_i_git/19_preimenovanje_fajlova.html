Preimenovanje fajlova

Kao i u slučaju brisanja fajlova, i preimenovanje fajlova možemo uraditi iz git-a, ili ručno pa onda to prijaviti git-u. Ako to uradimo ručno, git će nam prijaviti da smo naš fajl obrisali, i onda dodali novi fajl koji on još ne prati.

Da bismo dodali ove izmene, pisaćemo:

MINGW32:~

$ git add preimenovan_fajl.txt

$ git rm stari_naziv.txt

$_

Ovim su obe izmene prebačene u indeks, i u tom trenutku će ih git uporediti, i primetiti da je u pitanju samo preimenovanje:

MINGW32:~

$ git status
On branch master
Changes to be committed:
(use "git reset HEAD <file>..." to unstage)

renamed: stari_naziv.txt -> preimenovan_fajl.txt

$_

Git će registrovati sličnost između fajlova, čak i ako smo fajl izmenili nakon poslednjeg komita, a pre preimenovanja.

Što se gita tiče, promena naziva fajla, ili pomeranje fajla na drugu lokaciju u strukturi foldera koje on prati, su jedna te ista stvar. Tako UNIX to radi. Tako da ako želimo da promenimo naziv fajla kroz git, ili da ga prenesemo na neku drugu lokaciju, koristićemo istu komandu:

MINGW32:~

$ git mv fajl_za_preimenovanje.txt novi_naziv.txt

$ git status

renamed: fajl_za_preimenovanje.txt -> novi_naziv.txt

$_

Analogno ako hoćemo da pomerimo fajl u folder moj_direktorijum., pisaćemo:

MINGW32:~

$ git mv fajl_za_preseljenje.txt moj_direktorijum/fajl_za_preseljenje.txt

$ git status

renamed: fajl_za_preseljenje.txt -> moj_direktorijum/fajl_za_preseljenje.txt

$_

Pošto UNIX ne pravi razliku između preimenovanja, i preseljenja fajla, to mi možemo u isto vreme i preseliti fajl, i preimenovati ga.
