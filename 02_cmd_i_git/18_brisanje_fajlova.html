Brisanje fajlova

Ako smo u nekom trenutku ručno obrisali fajl koji je git pratio, i nakon toga uradimo $ git status, dobićemo sledeći odgovor od git-a:

MINGW32:~

$ git status
On branch master
Changes not staged for commit
(use "git add/rm <file>..." to update what will be committed)
(use "git checkout -- <file>..." to discard changes in working directory)

deleted: obrisan_fajl.txt

$_

Git nam govori da je fajl koji je pratio, obrisan, i šta da uradimo kako bi prestao da ga prati.

MINGW32:~

$ git rm obrisan_fajl.txt
On branch master
Changes to be committed
(use "git reset HEAD <file>..." to unstage)

deleted: obrisan_fajl.txt

$_

Sada kad uradimo commit, git će obrisati ovaj fajl iz svoje memorije fajlova koje prati.

Ovo očigledno radi, ali, zašto bismo gubili vreme da ručno brišemo fajl kad to isto možemo uraditi direktno iz git-a:

MINGW32:~

$ git rm fajl_za_brisanje.txt
rm fajl_za_brisanje.txt

deleted: fajl_za_brisanje.txt

$_

Što će ujedno da obriše fajl iz našeg direktorijuma, i da updejtuje git da ovaj fajl više ne prati. Ovde treba imati na umu da je u pitanju UNIX brisanje fajla, te da on nije prebačen u recycle bin, već je kompletno, načisto, sasvim obrisan.

Osim što i dalje postoji u repozitorijumu, tako da ako onda shvatimo da smo obrisali pogrešan fajl, i dalje ga možemo odatle vratiti.

Sada ostaje još samo da komitujemo izmene, čime se HEAD pomeriti, i fajl će ostati dostupan samo u prošlosti gita.
