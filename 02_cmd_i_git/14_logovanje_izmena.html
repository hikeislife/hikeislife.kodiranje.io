Pregledanje Komit loga

Ako postoji ikakva zabuna, log se u ovom kontekstu koristi sa značenjem "žurnal", "dnevnik", ili "poslovna knjiga". Nema nikakve veze sa "balvanima", "logaritmovanjem", ili logovanju na servere.

Da bismo izlistali sve prethodne izmene, koristićemo:

MINGW32:~

$ git log
commit d2f48781097e04ccc9d7785439a005e1cca63adc
Author: Ime Autora <email adreasa>
Date: Tue Aug 7 01:26:44 2018 +0200

Moj prvi komit

$_

Git log će izlistati nekoliko poslednjih komitova, a ako želimo da vidimo starije, možemo ih otvoriti pritiskom na "space bar". Enter bi takođe otvarao novi red loga, ali space otvara cele stranice. Ovaj log može izlistati sve izmene u nazad do prve, koliko god da ih ima.

Što se ispisa tiče, onaj džumbus slova i brojeva, je jedinstveni id broj našeg komita. Da, naš komit biva hašovan, tako nakon što nešto komitujemo, ne možemo se vratiti i krišom ispraviti grešku, jedino možemo ispraviti grešku, i ponovo je komitovati, sada pod novim id brojem.

U sledećem redu, git ispisuje vaše ime, i email, koje smo podesili ovde. Slede datum, i poruka koju smo upisali pri tom komitu.

Da biste izašli iz git loga, pritisnite Q na tastaturi

Kada budete imali veliki broj komitova, možda ćete hteti da izlistate samo nekoliko posledjih, tada ćete koristiti oznaku -n, i broj željeih komitova:

MINGW32:~

$ git log -n 2

commit a005e1cca63adcd2f48781097e04ccc9d7785439
Author: Ime Autora <email adreasa>
Date: Tue Aug 7 01:34:14 2018 +0200

Moj drugi komit

commit d2f48781097e04ccc9d7785439a005e1cca63adc
Author: Ime Autora <email adreasa>
Date: Tue Aug 7 01:26:44 2018 +0200

Moj prvi komit

$_

Slično, možete limitirati log po datumu:

MINGW32:~

$ git log --since=2018-08-07

$ git log --until=2018-08-07

U kom slučaju se datum piše po američki, u yyyy-mm-dd formatu. Ako koristite i --since i --until odjednom, možete precizirati period u kom vas zanimaju komitovi

Takođe je moguće izlistati komitove nekog autora, mada to više ima smisla koristiti kada niste jedini koji rade na projektu

MINGW32:~

$ git log --author="ime"

U navodnicima ili bez njih, i ne morate precizirati celo ime, dovoljno je ukucati samo deo imena.

Možete pretragu vršiti i po komit porukama, opet korišćenjem samo ključne reči, umesto cele poruke:

MINGW32:~

$ git log --grep="prvi"

Što bi izbacilo komitove čije poruke sadrže reč "prvi". Grep je skraćenica od "Global regular expression".

Ako uradite git log na folderu koji ste tek inicirali, ali niste još ništa komitovali, git će vam dati grešku:

MINGW32:~

$ git log

fatal: bad default revision 'HEAD'
