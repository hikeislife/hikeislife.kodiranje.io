Git prvi koraci

Instalacija Git-a

Da biste instalirali Git, skinite ga sa njihovog matičnog sajta, ovde. Kliknite na ilustraciju zelenkastog kompjuterskog monitora.

U pitanju je .exe fajl, tako da nema nekog dovijanja kako ga instalirati. Možete slobodno da ispratite korake za instalaciju bez ikakvih izmena, Git će se instalirati sa podešavanjima za vaš operativni sistem.
Ako ste na windowsu, možda ćete primetiti jedan ekran gde vas instalator pita šta želite da radite sa en: Line endingskrajevima redova. Ovde je poenta u tome da kompjuteri insertuju neki simbol na kraj reda kada vi pritisnete enter, da bi znali kako to da interpretiraju. Windows insertuje jedan simbol, a UNIX drugi. Pošto je softver originalno razvijan za UNIX sisteme, a vi sad planirate da ga koristite na windozi, on vas pita šta da radi s tom razlikom. Prva i podrazumevana opcija je da pri download-u fajla, konvertuje UNIX u windows simbole, a pri uploudu, da konvertuje win simbole u UNIX. Ovo je odgovarajući izbor, i možete nastaviti instalaciju.

Pri instalaciji Git-a, instaliraće vam se i Git Bash, koga ćete moći da otvorite iz start menija, a koji je UNIX okruženje, znači nešto kao Git UNIX. To će reći da će ovaj softver da oponaša UNIX na vašem windows kompjuteru i u njemu će raditi i UNIX, i windows komande. Moći ćete da koristite koje god su vam draže :)

Npr, možete izlistati sadržaj foldera po windowsu korišćenjem komande "dir", ili po UNIX-u, korišćenjem "ls -la". Svakako isprobajte obe, a naravno koristićete šta god vam ima više smisla.

MINGW32:~

$ ls

$ ls -la

$ dir

Ako ih isprobate i u komandnoj liniji, videćete da ls komande neće raditi. Zar to nije cool, možete naučiti malo UNIX-a bez da morate da menjate celo svoje radno okruženje.

Oznaka -la govori bash-u da nam pokaže i skrivene fajlove i foldere. Ovi fajlovi i folderi tipično imaju tačku ispred naziva, kao npr .git

Ako želite da vidite u kom folderu je Git Bash instaliran, ukucajte:

MINGW32:~

$ which git
/mingw32/bin/git

$ git --version
git version 2.8.1.windows.1

$ git --version komanda će nam reći koju verziju Gita imamo instaliranu, ako nas recimo za godinu dana bude zanimalo da li treba da je upgrejdujemo.

Podešavanja Git-a

Git čuva svoja podešavanja na tri različita mesta, u zavisnosti od toga koliko široko želimo da primenimo ta podešavanja.

  • Sistemska podešavanja su najšira i biće primenjena na svakog korisnika kompjutera. Svaki korisnik može da izmeni ova podešavanja za sebe, ali ovo će biti default, inicijalna podešavanja. Ova podešavanja se retko koriste, a mogu se naći u:
    • /etc/gitconfig folderu na UNIX-u
    • Program Files/Git/etc/gitconfig folderu na windowsu
  • podešavanja na nivou korisnika ćete naći u
    • ~/.gitconfig u vašem Home folderu na UNIX-u
    • Documents and Settings/korisničko ime/ folderu na windowsu
    u fajlu pod nazivom .gitconfig, bez extenzije.
  • Treći način da git čuva podešavanja, je na nivou projekta. Ova podešavanja će biti unutar foldera u kome se projekat nalazi, u folderu .git. U ovom sl, fajl se naziva samo config, bez extenzije.

U opštem slučaju, podešavanja na nivou korisnika imaju najviše smisla, osim ako nemate neka specifična podešavanja koja želite samo za neki projekat. Sistemska podešavanja, posebno za individualne korisnike, nemaju mnogo smisla. U svakom slučaju, ove fajlove možete otvoriti i pomoću tekstualnog editora, i tako ih izmeniti. Naravno, uvek možete menjati podešavanje iz Bash alata.

MINGW32:~

$ git config --system [željeno podešavanje na nivou sistema]

$ git config --global [željeno podešavanje na nivou korisnika]

$ git config --local [željeno podešavanje na nivou projekta]

Podešavanje korisničkog imena
MINGW32:~

$ git config --global user.name "Ime i Prezime"

Podešavanje email adrese

Za email adresu, ako koristimo github, umesto da koristimo svoju pravu adresu, koristimo generičku koju nam github daje, u suprotnom bi zlonameroboti mogli da je nađu, u roku od 2 minuta, i da nas zatrpaju spamovima.

MINGW32:~

$ git config --global user.email "vaše-github-korisničko-ime@users.noreply.github.com"

Kasnije, kad budemo više pričali o samom github-u, videćete kako da podesite github da prima "komitove" samo sa ove adrese, kako ne biste slučajno obznanili svoju email adresu svetu.

Kako proveriti svoja podešavanja

da biste videli svoja dosadašnja podešavanja, možete ukucati bilo koju od ove dve komande:

MINGW32:~

$ git config -l

$ git config --list

Ako želite da proverite specifično podešavanje, možete npr ukucati:

MINGW32:~

$ git config user.name
Vaše Ime

$ git config user.email
vaša@email.adresa

Ako ste pratili dosadašnje korake, već ste u svom korisničkom folderu na windowsu, a ako ste malo švrljali, možete se u njega vratiti komandom cd ~. Sada u ovom folderu možete videti svoj .config fajl pomoću:

MINGW32:~

$ cd ~

$ cat .gitconfig

Koja će vam otvoriti vaš tekstualni fajl sa podešavanjima, direktno u git bash-u.

Podešavanje podrazumevanog editora

nekada će vam git tražiti da nešto editujete u tekst editoru, pa je najbolje odmah podesiti podrazumevani editor na onaj koga najviše koristite:

MINGW32:~

$ git config --global core.editor "ime vašeg editora -wl1"

$ git config --global core.editor "'c:/Program Files/sublime text 3/subl.exe' -wl1"

Oznaka -w govori gitu da treba da sačeka da završite s editovanjem, a l1 mu govori da otvori fajl na liniji 1, dakle na početku fajla.

Dodajte boje u Git bash

Ovo je čisto opciono, ako želite, git može da vam prikazuje sav tekst monohromatski. Ako pak želite da git oboji određene informacije kako bi bile lakše uočljive:

MINGW32:~

$ git config --global color.ui true

Konačno počistite za sobom

Probajte ove dve komande, jedna će počistiti ekran, a druga će samo skrolovati sav dosadašnji tekst izvan vidnog polja, pa šta vam se više sviđa:

MINGW32:~

$ clear

$ reset
