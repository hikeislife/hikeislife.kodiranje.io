git status

U bilo kom trenutku, ako u bash ukucate git status i pritisnete enter, on će vam na ekranu ispisati odnos između vašeg radnog direktorijuma, indeksa, i repozitorijuma.

MINGW32:~

$ git status
On branch master
Your branch is ahead of 'origin/master' by 3 commits.
(use "git push" to publish your local commits)
Changes to be commited:
(use "git reset HEAD <file>..." to unstage)

modified: fajl.txt

Changes not staged for commit:
(use "git add <file>..." to update what will be commited)
(use "git checkout -- <file>..." discard changes in working directory)

modified: file1.txt
modified: file2.txt

$_

Ok, u opštem slučaju, $ git status vam ne bi izbacio ovoliku poruku. Ali hajde da vidimo šta tu sve ima.

Prva rečenica nam poručuje na kojoj smo grani. Do sada nismo pričali o granama, mada jesmo pomenuli forking, tako da ćemo ovo ostaviti za kasnije.

Druga rečenica nas obaveštava da je naša grana (master) ispred origin/master-a. Origin master je repozitorijum na github-u, o kome takođe još nije bilo reči, ali ono što nam git ovde govori je da smo uradili tri lokalna komita od kad smo zadnji put updejtovali origin/master. Dalje nam poručuje koju komandu da koristimo da sinhronizujemo ova dva repozitorijuma. I o ovome će biti više reči kasnije.

Zatim nam git navodi fajl.txt kao fajl koji je u indeksu, ali nije komitovan. Takve fajlove će nam prikazivati zelenom bojom.

Dva crvena fajla su oba u radnom direktorijumu, ali nisu dodata u indeks. "Changes not staged for commit:" je git poruka iznad ova dva fajla, dakle ako u ovom trenutku uradimo git commit, ova dva fajla neće biti komitovana. Git nam takođe govori kako da prebacimo ta dva fajla u indeks kako bi mogla da budu komitovana:

(use "git add <file>..." to update what will be commited)

O checkout-u će kasnije biti više reči.

HEAD

Jednu poruku gita smo preskočili:

(use "git reset HEAD <file>..." to unstage)

Ova poruka nam govori kako da fajlove koji su u indeksu izbacimo iz indeksa bez da ih komitujemo.

HEAD koga treba da resetujemo je kao igla gramofona, ili hard diska, naš trenutni položaj. Ako resetujemo "glavu", to znači da smo vratili iglu unazad. Svaki put kad nešto komitujemo, glava se pomera na taj novi komit. Kada promenimo granu na kojoj smo, glava se menja na poslednji komit na toj glavi. Ako se prebacimo na stariju verziju projekta, "glava" se seli na taj komit.

Git updejtuje "glavu" pri svakoj promeni.

Ako pogledate u svoj .git folder, videćete u njemu fajl HEAD bez ekstenzije. Ovaj fajl možete otvoriti pomoću tekst editora, i u njemu ćete videti samo jednu adresu:

ref: refs/heads/master

Ova adresa je relativna u odnosu na .git folder. U njemu ćete videti refs folder, a u njemu heads folder, i u njemu konačno fajl master, opet bez ekstenzije, koji opet možete otvoriti tekstualnim editorom.

Sadržaj ovog fajla, jeste haš našeg poslednjeg komita na datoj grani, odnosno komita na koji smo se vratili. To je kako git zna gde se u kom trenutku nalazimo, tako što u jednom fajlu drži ispisan trenutni haš.

Već je bilo reči da sadržaj .git foldera ne treba da menjate. Gledajte do mile volje, ali ne pipajte :)

git status foldera koga git ne prati

Ako se nalazite u folderu koji git ne prati, i uradite $ git status, git će vam dati grešku:

MINGW32:~

$ git status

fatal: not a git repository (or any of the parent directories): .git

Ovaj zadnji deo, govori da git ne nalazi .git folder ni u trenutnom folderu, ni u nekom od njegovih roditelja, po čemu zna da ovo nije folder koga on prati.
